# Contractor Lead Conversion System — Demo Walkthrough Script V1

## Objective
Give Jason a simple script for walking a prospect through the dashboard and explaining the offer.

The goal of the demo is not to show every feature.
The goal is to:
- make the pain obvious
- make the solution feel simple
- show what the client sees
- show what Jason sees
- make the next step easy

---

# 1. Core Rule
Do not do a long software tour.

The demo should answer 3 questions:
1. what problem does this solve?
2. what does it look like for the contractor?
3. how easy is it to add?

---

# 2. Demo Length
Best demo length:
**7 to 12 minutes**

If it goes much longer, you are probably talking too much.

---

# 3. Best Demo Opening
"The main problem this solves is pretty simple: if you’re already paying for leads or making the phone ring, missed calls and slow follow-up can quietly waste a lot of that spend.

What we built is a system that helps recover those missed opportunities, qualify the lead, and make it easier to move the right people toward booking."

That is the best opening.

---

# 4. What to Show First
## Show the client dashboard first
Why:
- it shows clear value fast
- it’s easier to understand
- it answers the buyer’s question: “what would I actually see?”

---

# 5. Client Dashboard Talk Track
## Overview screen
"This is the client side. It shows the leads coming in, missed calls, recovered leads, qualified leads, booked estimates, and recent activity. So instead of guessing what happened to inbound leads, they can actually see it."

### Point out
- total leads
- missed calls
- recovered leads
- booked estimates
- recent activity

## Key line
"The point is visibility and faster recovery — not more software complexity."

---

# 6. Lead / Conversation View Talk Track
"This is where they can actually see what happened on an individual lead. A call comes in, the system follows up, the lead replies, qualification starts, and the team can see whether it turned into a real opportunity or not."

### Point out
- missed call event
- text thread
- qualification questions
- lead status
- urgency / booking intent

## Key line
"Instead of a missed call just disappearing, it becomes a trackable conversation."

---

# 7. Alerts View Talk Track
"If something looks like a hot lead, callback request, or urgent situation, it gets flagged so the business can respond faster."

## Key line
"This helps the right leads get attention instead of sitting buried."

---

# 8. Scorecard View Talk Track
"This is the monthly proof layer. It shows how many leads came in, how many missed calls were recovered, how many leads were qualified, and how many estimates were booked or moved forward."

## Key line
"This is how the client sees whether the system is actually doing useful work month to month."

---

# 9. Admin View Talk Track
After showing the client side, switch to admin.

"This is the operator side — this is what we use to manage the system across accounts, keep workflows healthy, review conversations, track onboarding, and make sure things are actually working."

### Point out
- client roster
- health / workflow status
- onboarding status
- docs / build log / playbooks
- operations controls

## Key line
"The client sees their business. We see the whole operating system behind it."

---

# 10. Explain the Implementation Simply
"The nice part is this sits on top of what they already do. It doesn’t require replacing their whole phone system or CRM on day one. It’s designed to be easy to add and easy to remove."

That reduces fear.

---

# 11. Pricing Talk Track
"For early clients, the setup is simple — $500 setup and $500 per month. That includes the missed-call recovery setup, qualification flow, dashboard access, alerts, and monthly reporting."

Then stop.
Do not overtalk pricing.

---

# 12. Close / CTA
## Best CTA
"If this looks useful, the next step is simple — we’d get a few details from you, set up the number and rules, test it, and launch it."

Alternative:
"If you want, we can set up a pilot and keep it simple."

---

# 13. Best Questions to Ask During Demo
Use these to keep them involved:
- "What usually happens if a call gets missed today?"
- "Do after-hours leads usually wait until the next day?"
- "Would visibility like this be useful on your side?"
- "Are most of your better leads coming in by phone?"

These help tie the demo to their business.

---

# 14. What NOT to Do
Do not:
- explain the whole backend
- explain AI models
- explain technical stack
- click through every screen too fast
- oversell future features
- talk longer than the prospect

This is a business conversation, not a developer tour.

---

# 15. Simple 60-Second Version
If you need a very short version:

"This system helps contractors recover missed inbound leads, qualify them faster, and get better visibility into what happens after a call comes in. The client sees the lead dashboard, conversations, alerts, and monthly scorecard. On our side, we manage the workflow and optimization so it actually keeps working."

---

# 16. Goal of the Demo
A good demo does **not** need to close the full deal on the spot.

A good demo should get one of these outcomes:
- serious interest
- pricing conversation
- pilot discussion
- onboarding discussion

That is enough.
